Давайте попробуем составить стратегию тестирования TRACKS API.

В качестве помощи при составлении вы можете использовать HTSM от Джеймса Баха.

Чтобы немного ограничить задачу, предлагаю вам сконцентрироваться на следующих вещах:
  1. Критерии качества (Quality Criteria) - какие критерии будут наиболее важными для API?
  2. Техники тестирования - какие техники тестирования мы включим в стратегию?
  3. Элементы продукта - из каких частей состоит продукт, какие функции он выполняет, с какими данными он работает?
  4. Автоматизация - будем ли мы автоматизировать тесты? Какие инструменты автоматизации будем использовать?

Для наглядности вы можете написать основные положения вашей стратегии на бумаге - в формате списка по пунктам или mind map.
